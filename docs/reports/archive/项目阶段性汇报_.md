# 面向半导体 FAB 机台组调度的两层强化学习框架
## 阶段性汇报

---

## 一、问题背景

半导体晶圆制造是典型的大规模、强不确定调度问题。本项目面向 FAB 机台组调度：给定一批待加工 Lot（晶圆批次）、若干**异型且柔性**的机台、工艺配方（PPID）以及腔体/Side 资源，需决定**把哪个 Lot、用哪个配方、在何时派往哪台机器**，并在两类不确定性下完成调度：

- **工件随机到达**——Lot 何时进入系统不可预知，仅有限前瞻窗内可见；
- **加工时间随机波动**——同一工序的实际耗时围绕标称值抖动。

同时必须满足三类强弱不同的要求：**阶段间 Q-time（材料队列时间）硬约束**（超时即整批报废，零容忍）、**优先级强偏好**、以及尽量提高**机台利用率**（唯一可真正权衡的软目标）。

该问题属强 NP-hard 的柔性流水车间调度。传统做法二者必居其一的短板明显：启发式规则实时但只看局部、全局性能弱；数学规划/元启发式全局但规模一大就解不动，且难以应对不确定性。

---

## 二、核心思路：两层解耦架构

本项目的关键判断是：**这个问题的决策天然分两层，且只有一层需要"学"。**

| 层 | 职责 | 方法 |
|----|------|------|
| **上层（学习 + 搜索）** | 决定"哪个工件、哪台机器、哪个配方、什么时机" | RL 派工（SAS）＋ VC-MCTS 搜索式预留 |
| **下层（固定规则）** | 工件派定后，其一批晶圆如何组批、走哪些腔体、各批时序，并估算完成时间分布 | 确定性组批/流水排程 + 蒙特卡洛 |

两层之间是**单向窄接口**：上层只把"工件—机器—配方"交给下层，下层只回传**完成时间分布 (μ, σ)** 与资源占用区间。下层被确立为**调度时序的单一真相源**——估时（`estimate`）与落地（`schedule_on_calendar`）共享同一 list-scheduling 核心，从根上消除了"上层自行重算批处理、与估时口径不一致"的隐患。

> 意义：把"需要智能权衡的部分"和"只需算准的部分"彻底分开，既缩小 RL 的学习负担，又保证时序估计一致、可执行。

## 三、创新点

本项目的目标不是"几个目标加权"，而是**严格分先后的三层字典序**，由候选池流水线 `mask → filter → score` **结构强制**（而非靠调权重）：

> **Q-time（硬约束）＞ 优先级（强偏好）＞ 利用率（唯一软目标）**

危险动作根本不进入智能体的可选集合：① Q-time → mask（移出动作空间）；② 优先级 → 过滤（仅在 mask 存活集内生效，绝不可能为抢优先级而踩穿 Q-time）；③ 利用率 → 打分。可讨价还价的取舍被收敛到唯一一层（利用率），上两层是"门槛"而非"权重"。**两个核心创新点正好坐落在最关键的两层上**——创新点 1 治理第 0 层（Q-time 硬约束），创新点 2 治理第 1 层（优先级）。

### 1. Q-time 链感知的联合机会约束 mask 〔第 0 层：把 Q-time 报废压到近 0〕

**问题**：把约束写进奖励做软惩罚原理上站不住——硬约束靠罚分劝不住，只要别处奖励够高，智能体仍会为利用率"买"一次违规。但即便把它做成 mask，还有两个更隐蔽的坑：

- **筛选对象**：一个工件单步内部要顺 chamber 序列走多个**阶段**（stage1→stage2→stage3），真实 Q-time 是**逐段的队列时间窗** `(1,2)`、`(2,3)`（共享 stage2、首尾相连，构成一条**链**），超任一段即整批报废。

**机制：让 mask 对齐真实耦合链，并做成联合机会约束。**

- **对齐真实链**：对每个候选**非破坏式试算**，用真实逐段时序判每一段窗口（`compute_q_time_violation`）——"屏蔽所筛 = 实际所罚"。
- **联合机会约束**：下层估时器输出完成时间分布 (μ, σ)，把"链上**任一段**违规"的概率压到 ε 以下（`P(∪ₖ 违窗ₖ) ≤ ε`）；安全裕度从拍脑袋常数变成自适应的 `z_ε·σ`（波动大的工件自动留厚垫）。两档实现：`chain`（单次带噪试算，抓单样本可见的违规）、`chain_joint`（K 次独立带噪试算估违规概率、连**噪声尾部**也屏蔽）。
- **窗外残差兜底**：窗外才到达 / 正态尾部这类当前不可预判的残差，用 **PPO-Lagrangian（CMDP，自适应"违规价格"λ）** 在训练期软压——**窗内硬挡、窗外软压**，分工清晰。

**为什么 novel**：通用 RL 调度论文把 Q-time 建在聚合量或逐段独立窗口上，**没人做"耦合链上的联合机会约束、且筛选与实际报废同口径"的 mask**。

### 2. 有限前瞻下的搜索式主动资源预留（VC-MCTS）〔第 1 层：为高优先级工件腾产能〕

**问题**：真实 FAB 中工件何时进入系统不可预知，高优先级工件可能稍后才到。主流事件触发式 RL（机器一空闲就从已到达工件里挑一个派出）**结构上无法表达"先不派、把产能留给一个即将到达的高优先级工件"**，只能被动响应。把预留做成可学习的智能体也不行——实证发现学习范式有**结构性顽疾**：预留奖励稀疏且与派工奖励重复计数；触发器为保覆盖率必须宽开，造成**类别极度不平衡**持续把策略推向"从不预留"；最致命的是贪心推理临门一脚——高优先级工件到达前那一下 `P(reserve)` 顶不过 0.5，再强的编码器也救不回来。这是学习范式本身的失败模式，非调参可解。

**机制：用搜索取代学习承担预留决策。** VC-MCTS 在每个决策点把"现在派 vs 留给未来"展开成可直接模拟比较的若干分支，按字典序目标选出最优边：

1. **ROP 当算力闸门挑候选**：以"宽触发 + 软评分（优先级落差 + 机器稀缺度 + ETA 近度）+ TopB 截断"筛出少量 `(机器 m, 未来工件 h)` 对，只回答"是否值得搜一搜"；无机会点的决策直接走贪心派工、近乎免费。
2. **构造根边**：搜索边 = `dispatch ∪ reserve(m,h) ∪ no_op`；`dispatch` 边的具体 `(lot,ppid)` 经 `dispatch_delegate` 交给训练好的 SAS——**"派哪个"由学习的 SAS 决定，"要不要为未来预留"由搜索决定**。
3. **每条边忠实仿真到底**：deepcopy 环境与预留台账（非破坏性），施加动作后走资源日历 commit 把 episode 跑到底；基策略与树策略同源（同走 Q-time 安全候选池、同尊重预留台账、同注入工艺噪声）。
4. **字典序选边 + no_op 护栏**：按 `Q-time 违规 → O2 → 利用率` 选最优边，并强制"只有 no_op 在 Q-time 上严格更优才允许空等"，防止退化为一直空等。

由此，**预留收益就是 reserve 分支与 dispatch 分支模拟出的 O2 之差，被精确算出，无需手工代理、也无需学一个概率**。被选中的 `reserve` 登记进预留台账（机器、目标工件、ETA、TTL），等目标工件到达后优先兑现、超时则释放——预留从模糊的 hold 变成有对象、有责任、有验收（命中/浪费）的决策。

**理论定位**：本方案是 DyRo-MCTS（ICLR 2025）的原理性推广——后者完全看不见未来到达、只能用钝的鲁棒性代理；本项目在有限前瞻窗内未来到达可见，于是"定向预留"从浪费变为最优。当前为 root-level 单层 rollout 比较，SAS 先验 `p(s,a)` 与多头 Critic 叶子估值接口已实现（默认关闭），作为升级为完整 MCTS 的下一步。

### 两点如何协同：一个决策点的完整流程

下图把创新点 1（Q-time 链 mask）与创新点 2（VC-MCTS 预留）串成一次决策：**搜索决定"派 / 留 / 等"，一旦决定"派"，再由三层字典序候选池决定"派哪个"**——预留与硬约束各管一段、互不越界。

```
每个决策点：选出一台空闲机器 m（最受约束优先）
        │
        ▼
   存在预留机会点 ROP ?  ──否──────────────────────┐
        │ 是（宽触发 + 软评分 + TopB 截断）           │
        ▼                                            │
  ┌──────────── 创新点 2：VC-MCTS 搜索 ──────────────┐│
  │ 根边 = reserve(m,h) | dispatch | no_op            ││
  │ 每条边 → 非破坏式 rollout 到底（基策略=SAS，同源） ││
  │ 按字典序  Q-time违规 → O2 → 利用率  选最优边       ││
  │ （no_op 护栏：仅当 Q-time 严格更优才允许空等）     ││
  └───────┬──────────────┬───────────────┬──────────┘│
          │ reserve       │ no_op         │ dispatch  │
          ▼               ▼               ▼           ▼
   登记预留台账      推进时间到     ┌─ 创新点 1：链感知机会约束候选池（顺序不可换）─┐
   m 冻结给 h        下一事件       │ 结构可行集                                    │
   待 h 到达兑现                    │   ① Q-time 链 mask（硬）：对每候选试算，按真实  │
   超时则释放                       │      阶段链 (1,2)/(2,3) 联合机会约束移出动作空间 │
                                    │   ② 优先级过滤（强）：仅在①存活的安全集内      │
                                    │   ③ 利用率打分排序（软）：唯一可让步维度        │
```

## 四、关键结果

**已建成并打通**：资源日历调度环境（dry-run/commit 保证可执行、可回滚）、下层单一真相源估时器、候选池三层约束流水线（含**链感知机会约束 mask**）、上层 SAS 多头 Critic + 向量奖励 PPO（含 Q-time 的 PPO-Lagrangian）、**VC-MCTS 在线预留规划器**（reserve/dispatch/no_op 搜索 + dispatch_delegate 接入 SAS + 预留台账 + ROP 闸门 + JSONL 可解释轨迹），以及多随机种子评测体系（自建 50 Lot 压力基准、`pressure` 与"高优先级晚到 `late_hi`"两类实例 + 经典规则基线）。

### 结果一（对应创新点 1）：链感知 mask 把 Q-time 报废压到近 0

为隔离 mask 本身的效果，用最简单的 FIFO 派工（不掺入任何 RL/搜索增益），在 `late_hi` 与 `pressure` 两实例、各 5 个工艺噪声种子上对比：

| 指标（FIFO, noise, 5 seed） | 聚合代理 mask（原） | **链感知 mask（本项目）** |
|---|---|---|
| Q-time 违规均值 ↓ | 13.6 | **0.4** |
| Q-time 尾部（最坏种子）↓ | 26 | **2** |
| Q-time 累计超时均值 ↓ | 11.25 | **0.14** |
| 机台利用率 | 0.821 | 0.812 |

**两实例数值完全一致**：Q-time 违规 **−97%**、尾部 **−92%**、5 个种子里 **4 个直接清零**，而利用率仅降 1%、O2 不退反进、无死锁。进一步开启 `chain_joint`（联合机会约束、对噪声尾部也屏蔽）可把剩下那个残留种子也清到 **0**。这直接验证：**Q-time 清不掉的根因是 mask 筛错了对象（聚合代理 ≠ 实际链窗口）；把 mask 对齐真实阶段链后，近乎免费地解决了硬约束**——正中第一优化目标。

### 结果二（对应创新点 2）：VC-MCTS 选择性预留同时压过全部经典规则

> **O2（优先级加权等待）定义。** 对每个工件 lot，其"等待" = 从到达到首次开工的时长 `max(0, start − arrival)`，再按该工件优先级加权；O2 是全部工件加权等待之和：
>
> ```
> O2 = Σ_lot  priority(lot) × max(0, start(lot) − arrival(lot))
> ```
>
> **越小越好**——它把"高优先级优先"从口号变成可比较的标量（高优先级被压在队列里越久、惩罚越重）。

`late_hi` 实例（工艺噪声，5 种子），完整模块（VC-MCTS+SAS）与五条经典规则同口径 head-to-head：

| 策略（late_hi, noise, 5seed） | O2 加权等待 ↓ | Q-time 违规数 ↓ | 利用率 |
|------|--------------|----------------|--------|
| FIFO（规则中 O2 最优） | 1584.0 | 13.6 | 0.819 |
| ATC / SPT（优先级敏感规则） | ≈1870 | 6.0 / 6.8 | 0.86 |
| **VC-MCTS + 规则派工** | 1245.6 | **0.0** | 0.830 |
| **VC-MCTS + SAS 派工（完整模块）** | **1240.4** | 1.2 | 0.800 |

完整模块 O2 比规则最优的 FIFO 再低 **−21.7%**、比优先级敏感的 ATC/SPT 低 **≈−34%**，同时把 Q-time 从两位数压到近 0——**在优先级与 Q-time 两条关键轴上同时超越全部五条规则**。对照实验：单纯把 SAS 用作派工（无预留）对 O2 几乎无改善（+4.5%），**证明增益来自"预留"这一结构、而非更强的派工**。该增益不限于刻意构造的 `late_hi`：在随机优先级的 `pressure` 上，VC-MCTS 同样把 O2 降约 **−23%**。

### 结果三（地基）：基础 SAS 派工已优于经典规则

压力基准、10 个随机噪声种子，训练后的 SAS 与最优规则对比：Q-time 违规 **6.0 → 1.4**（较最优规则减少约 4.3 倍）、利用率 **86.0% → 87.2%**——在两条真正区分策略的维度上同时取得最优，说明上层学习的派工地基本身已扎实，为其上的两个创新点提供了可归因的基线。
